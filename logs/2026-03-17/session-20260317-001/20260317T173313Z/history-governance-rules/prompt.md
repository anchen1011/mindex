# Prompt

This project needs to follow two major rules:

1. Comprehensive Testing:
All work must undergo detailed testing. A task is not considered complete simply because the code is finished; it requires explicit tests and documented results. These results should be recorded in the logs with clear distinctions regarding what was executed and whether it succeeded. Every task's completion should be measured by the successful completion of its corresponding tests.

2. Git Management and Pull Requests:
All changes must be actively committed to Git. Regardless of the type of code, everything must be managed under Git. Commits should be made systematically as each sub-task is finished. If the repository is hosted on GitHub, all contributions (especially those submitted by AI) must be handled through Pull Requests (PRs). Direct pushing to the main branch is not allowed; every change should go through a PR before being merged.
